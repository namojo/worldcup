# fifa-ui-theme — FIFA풍 인터랙티브 UI 시스템

FIFA 게임/방송 중계 그래픽의 분위기를 단일 HTML(바닐라 JS)로 구현하기 위한 디자인 토큰·컴포넌트·인터랙션 지침. 목표는 "스타디움의 긴장감 + 데이터의 명료함".

## 미감 방향 (왜)
FIFA UI의 핵심은 **어두운 경기장 위로 떠오르는 발광하는 정보 레이어**다. 정보는 방송 그래픽처럼 굵고 명료하게, 배경은 깊고 차분하게, 강조는 네온처럼 날카롭게. 밋밋한 회색 카드와 기본 폰트는 즉시 "비FIFA"로 읽히므로 피한다. 절제된 모션이 고급감을 만든다 — 모든 것이 움직이면 아무것도 강조되지 않는다.

## 디자인 토큰
```css
:root{
  /* 스타디움 다크 베이스 */
  --bg-0:#070b14; --bg-1:#0d1424; --bg-2:#141d33; --surface:#16203a;
  --line:rgba(255,255,255,.08);
  /* 네온 강조 (피치 그린 + 일렉트릭) */
  --accent:#00ff87;        /* 메인: 피치 그린/민트 */
  --accent-2:#00d4ff;      /* 보조: 일렉트릭 시안 */
  --accent-3:#ff2d6f;      /* 라이브/경고: 핫 핑크 */
  --gold:#ffd24a;          /* 트로피/득점왕 */
  --text:#eaf1ff; --text-dim:#9fb0cf;
  --grad-hero:linear-gradient(135deg,#00ff87 0%,#00d4ff 100%);
  --grad-card:linear-gradient(160deg,rgba(0,255,135,.06),rgba(0,212,255,.03));
  --shadow:0 18px 50px rgba(0,0,0,.55);
  --radius:16px;
}
```
- 강조색은 **포인트로만**. 큰 면적은 다크 베이스, 강조는 테두리·텍스트·작은 글로우에.
- 라이브 경기는 `--accent-3`(핫 핑크) 맥동, 트로피/득점왕은 `--gold`.

## 타이포그래피
- 디스플레이/숫자: 굵고 압축된 스포츠 산세리프 — `'Oswald','Archivo Narrow','Anton', system-ui` (CDN 실패 대비 system-ui fallback 필수). 스코어·팀코드는 무겁게(700~900), 약간의 letter-spacing.
- 본문: `'Inter', system-ui, sans-serif`.
- 위계: **스코어(가장 큼) > 팀명 > 경기 메타(시간·경기장)**. 숫자는 `font-variant-numeric: tabular-nums`로 정렬.

## 핵심 컴포넌트 패턴

### 1. Hero / 스코어보드 헤더
대회명 + 개최국 + 카운트다운(다음 경기까지) + 라이브 인디케이터. 배경에 미묘한 피치 라인(반복 그라데이션) 또는 스타디움 글로우. 그라데이션 텍스트(`--grad-hero`)로 제목 강조.

### 2. 경기 카드 (status 3종 분기 — 가장 중요)
같은 카드가 데이터의 `status`에 따라 다르게:
- **finished**: 큰 스코어 양옆에 국기+팀코드, 하단에 "하이라이트 보기" 토글. 승팀 측에 미세한 글로우/볼드.
- **live**: 맥동하는 빨간 `● LIVE` 뱃지, 현재 스코어, 테두리 핫핑크 글로우, 진행시간(예 67').
- **scheduled**: 스코어 자리에 `VS`와 킥오프 시간/카운트다운, 차분한 톤.
- 호버 시 살짝 떠오르고(`translateY(-4px)`) 테두리 발광.

### 3. 필터/탭 바
`전체 · 🔴 진행중 · ⏰ 예정 · ✅ 종료 · 조별(A~L)`. 활성 탭은 알약형 + 네온 밑줄/글로우. 클릭 시 카드가 부드럽게 필터링(fade/stagger).

### 4. 조별 순위표
조별 standings 테이블. 승점 기준 정렬, 상위 2팀(진출권)에 좌측 네온 바 표시. 컬럼: 팀 / 경기 / 승 / 무 / 패 / 득실 / 승점. tabular-nums.

### 5. 하이라이트 모달/확장
경기 카드의 하이라이트 토글 → 모달 또는 인라인 확장으로 핵심 장면 불릿·스토리라인 표시. 배경 블러(backdrop-filter), 등장 애니메이션.

### 6. 토너먼트 브래킷(선택)
16강 이후가 있으면 가로 스크롤 브래킷. 미정 경기는 placeholder 노드.

## 인터랙션 / 모션 원칙
- 전환은 `cubic-bezier(.2,.7,.2,1)` 200~350ms. 과하지 않게.
- 진입 시 카드 stagger fade-up(IntersectionObserver). 60fps 느낌(transform/opacity만 애니메이트, layout thrash 금지).
- 라이브 뱃지 맥동(`@keyframes pulse`), 카운트다운 1초 틱(setInterval).
- 호버: 카드 lift + 테두리 글로우, 버튼 채도 상승.
- 숫자 카운트업(스코어/득점 등장 시) 같은 디테일이 FIFA 감성을 살린다.
- `prefers-reduced-motion` 존중.

## 구현 규칙 (단일 HTML)
- 한 파일: `<style>` + `<body>` 마크업 골격 + `<script>`. 데이터는 상단 `const WORLD_CUP_DATA = {...}`.
- **데이터 주도**: 경기/조/순위 마크업은 JS로 `WORLD_CUP_DATA`를 순회해 생성. 정적 하드코딩 금지.
- 폰트는 Google Fonts CDN `<link>` + system fallback. 국기는 이모지(오프라인 렌더).
- 반응형: 모바일 1열 → 데스크톱 그리드(`auto-fill, minmax(300px,1fr)`). 헤더/탭 모바일 대응.
- 접근성: 시맨틱 태그, 버튼은 `<button>`, 모달 포커스 트랩·ESC 닫기, 대비 WCAG AA 지향.

## 안티패턴 (AI 슬롭 — 피할 것)
- 밋밋한 회색/흰색 카드, 기본 폰트, 의미 없는 보라 그라데이션.
- 모든 요소에 그림자·둥근모서리 남발.
- 강조색을 큰 배경 면적에 도배(눈 피로).
- 데이터와 무관한 가짜 차트/장식. 스코어보다 장식이 큰 위계 역전.

## 사용자 확정 요구사항 (반드시 반영)
- **팀명은 한글로.** 코드(MEX) 옆/대신 한글 국가명(멕시코)을 노출. 데이터의 영문명을 한글로 매핑하는 단일 `TEAM_KO` 표(코드→한글)를 두고, 경기 카드·순위표·득점왕·카운트다운 전부에 적용.
- **모든 시각은 한국시간(KST).** 데이터는 현지(ET) 기준으로 저장하되, 화면 표기는 `Intl.DateTimeFormat(timeZone:'Asia/Seoul')`로 변환해 보는 사람의 PC 시간대와 무관하게 항상 KST로. "KST" 라벨 명시.
- **다음 경기 동시 다발 처리.** 카운트다운은 가장 이른 "미래" 킥오프 시각을 찾고, **같은 시각의 경기를 모두** 표시(동시 N경기 표기). 단일 경기만 가정하지 말 것.
- **가짜 상태 금지.** 실제 live 경기가 없으면 "LIVE" 같은 가짜 뱃지를 만들지 말고, 데이터 기반의 의미 있는 상태(진행 경기 수, 단계 등)만 노출.
- **유튜브 하이라이트 링크.** 종료 경기마다 하이라이트 영상 링크 제공. 직접 영상ID 하드코딩은 삭제/오링크 위험이 크므로, 기본은 경기별 유튜브 검색 딥링크(`youtube.com/results?search_query=...`)를 사용하고, 데이터에 `youtube` 필드가 있으면 그 URL을 우선.
- **토너먼트 로드맵 상시 준비.** 조별리그만 있을 때도 32강~결승 라운드를 로드맵으로 미리 노출(날짜·경기수·"대진 확정 예정"). `matches[]`에 녹아웃 경기(stage=Round of 32 등)가 추가되면 진행률·상태가 자동 갱신되고 경기 카드 그리드에도 자동 등장하도록 데이터 주도로.
